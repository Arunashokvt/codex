# Privacy Policy

**Last updated:** April 29, 2026

This Privacy Policy explains how the **GitLab MR Review** plugin (the “Plugin”) handles information.

## 1. Scope

This policy applies to use of the Plugin in Codex environments.

## 2. What the Plugin Accesses

When you run the Plugin, it may access:

- Current git branch name (from your local repository).
- Merge request metadata (for example MR ID, title, URL, state).
- Merge request discussions/comments and related fields.
- Local file contents in your working repository when applying fixes.

## 3. How Data Is Used

Data is used only to:

- Find the MR for the current branch.
- Fetch and summarize review comments.
- Support user-selected comment fixing workflows.
- Produce outputs requested by the user.

## 4. Data Storage and Retention

- The Plugin is designed to operate in-session and does not include a separate hosted backend.
- The Plugin does not intentionally persist personal data outside normal Codex/tool execution output and local workspace files.
- Temporary command output may exist briefly in runtime memory or temporary files during execution.

## 5. Data Sharing

The Plugin does not sell personal data.

Data may be processed by third-party systems you use directly, including:

- GitLab APIs/services accessed through `glab`.
- Codex/OpenAI product infrastructure used to run your session.

Their own privacy policies and terms apply.

## 6. Security

Reasonable technical steps are used within the Plugin workflow, but no method of transmission or storage is 100% secure.

## 7. Your Responsibilities

You are responsible for:

- Reviewing organization policies before using the Plugin on sensitive repositories.
- Managing credentials/tokens used by `glab`.
- Avoiding use on data you are not authorized to process.

## 8. Children

The Plugin is not directed to children under 13.

## 9. Changes to This Policy

This policy may be updated from time to time. The “Last updated” date will be revised when changes are made.
